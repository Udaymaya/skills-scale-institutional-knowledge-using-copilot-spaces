# OctoAcme — Role-Responsibility Mapping Checklist

## Purpose
Provide a quick-reference checklist to ensure all critical project responsibilities are assigned and owned by a named role. Use this during project initiation and planning to reduce gaps and ambiguity.

---

## Project Initiation

| Responsibility | Owner Role | Assigned To | Status |
|---|---|---|---|
| Define problem statement and objectives | Product Manager / Product Owner | | ☐ |
| Identify stakeholders and communication needs | Project Manager | | ☐ |
| Draft project charter or one-pager | Project Manager | | ☐ |
| Confirm resource availability | Project Manager | | ☐ |
| Capture initial risks and dependencies | Project Manager / Business Analyst | | ☐ |

---

## Requirements & Planning

| Responsibility | Owner Role | Assigned To | Status |
|---|---|---|---|
| Gather and document business requirements | Business Analyst | | ☐ |
| Prioritize and groom product backlog | Product Owner | | ☐ |
| Translate requirements into user stories | Business Analyst / Product Owner | | ☐ |
| Define acceptance criteria for stories | Product Owner / QA Lead | | ☐ |
| Establish sprint plan and milestones | Project Manager | | ☐ |
| Identify technical risks and mitigations | Developers / Project Manager | | ☐ |

## Notes
- Update the **Assigned To** column at project kickoff.
- Review this checklist at each phase gate to confirm ownership.
- Escalate unassigned responsibilities to the Project Manager immediately.
